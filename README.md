# Sistema de Gerenciamento de Tarefas 

Um controlador de tarefas robusto desenvolvido em Java para organizar a rotina diária. O projeto foi estruturado com foco na aplicação prática dos fundamentos de Programação Orientada a Objetos (POO), manipulação de coleções e tratamento de erros.

---

## 🚀 Funcionalidades

- **Gerenciamento de Tarefas:** Adicionar e remover tarefas de forma dinâmica.
- **Controle de Status:** Marcar tarefas como concluídas ou pendentes.
- **Filtro por Período:** Visualizar resumos de tarefas específicas baseadas em dias da semana.
- **Validação de Entradas:** Tratamento de erros para garantir que o usuário interaja apenas com dados válidos.

---

## 🛠️ Conceitos de Engenharia de Software Aplicados

Para construir este sistema além da sintaxe básica, foram aplicados os seguintes pilares:

* **Orientação a Objetos & Encapsulamento:** Modelagem de objetos com atributos privados e métodos acessores bem definidos para garantir a segurança e organização dos dados.
* **Estruturas de Dados (`java.util.ArrayList`):** Uso de listas dinâmicas para armazenar e manipular os fluxos de tarefas com eficiência.
* **Interação via Console (`java.util.Scanner`):** Criação de uma interface de linha de comando amigável para inputs do usuário.
* **Tratamento de Exceções (`try-catch`):** Implementação de lógica de resiliência. Caso o usuário insira um dia da semana inválido ou um comando inesperado, o sistema trata o erro e solicita a informação novamente de forma segura, sem interromper a execução do programa.
